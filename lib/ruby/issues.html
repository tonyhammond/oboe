Oboe ~ Open Issues

All issues recorded below are still open and subject to change. They have not yet been approved by the Committee.

EBNF

This resolver makes use of the following modifications from the published EBNF:
  • The version number "Z39.00-00" is used in place of "Z39.NN-NN". Not addressed is the notion of using the version number as a flag to indicate the type of encoding: name/value or XML.

  • The namespace ID "_ri" is used in place of the "local" namespace ID. Rationale is to acknowledge that this is a foreign resource identifier and to maintain a resource identifier progression ( "uri" - uniform, "ori" - openurl, and "_ri" - foreign).

  • Format identifiers now use identifiers - rather than names - within any of the resource identifier namespaces.

  • Inline metadata is flagged with a "ref_val" parameter, which complements the "ref_ptr" parameter that identifies a remote instance. The "ref_val" parameter is merely declarative and does not need to take any value although if one is present "*" is suggested. This is the missing descriptor key which previously was inferred by the presence of inline metadata.

  • Metadata format identifiers are now of the form "ref_val_fmt" for inline metadata format identifier, and "ref_ptr_fmt" for remote metadata format identifier. The second suffix ("_fmt") marks off format identifier from descriptor.

  • Private data descriptors have key suffix "dat" instead of "pid". Rationale is that private identifiers - rather than data - are now accommodated under the identifier descriptor with a foreign namespace ID ("_ri"). The same suffix is used for the context-object private data.

XML

  • The XML uses element name "<metadata-by-val>" rather than "<metadata>" to complement element name "<metadata-by-ref>".
  • The element name "<context-object-private-data>" is now more simply "<private-data>".
