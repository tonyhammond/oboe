Oboe ~ About This Resolver

Summary

This OpenURL resolver parses, validates and services an OpenURL.

Parsing

OpenURL parameters are read in batch mode from a querystring or data post. In user mode parameters are read from the web form. Remote parameters can be fetched over the network from a property list using a context-pointer.

Validation

The validation process includes checks on the administration version, the presence of a referent entity descriptor, legal identifier namespace IDs and schemes, and legal metadata format identifiers and keywords. A more complete summary of checks is listed under features.

Services

The default service of this resolver is to render this OpenURL as a property list. Other renditions (Excel, Querystring, RDF, XML) can also be selected.

Further services are available through the use of service directives which are implemented by using identifiers of service-type. Examples include: (no)strict, (no)validate, (no)refs, trace, forward (to another service component). See directives for a fuller listing.
