Oboe ~ Resolver Features

This OpenURL validating resolver has the following features:

  Context Import   URL querystring; property list fetched from a context-object URL reference
  Context Export   Property list; Excel spreadsheet; URL querystring ; RDF description ; XML document
  Validation   Validation includes:

   - Check for admin version and for legal version string
   - Check for legal admin char_set value if present
   - Import context-object from property list encoding
   - Check for referent entity
   - Check for legal identifier namespace IDs
   - Check for referrer entity if foreign namespace ID used
   - Check for legal identifier schemes
   - Check for well-formedness of uri identifiers
   - Check for legal metadata format identifiers
   - Check for legal metadata format identifier terms
   - Check on entity cardinalities - only one of each type allowed
   - Check on descriptor cardinalities - only one of each type allowed unless identifier
   - Check on restriction of metadata descriptor type to qualify referent entity only

   Still to do:

   - Import context-object from XML encoding
  Help & Hints   Help and hints are given on validation error pages
  Example Contexts   EBNF example contexts are bundled and randomly selected from if no context is presented
  Directives   Service directives include control of validation process and rendering options
  Legacy Support   Support v.0.1 by upgrading to v.1.0 (work in progress)
