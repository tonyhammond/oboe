Oboe ~ Service Directives

The following service directives can be attached to the OpenURL to modify the resolution behaviour:

  srv_id=_ri:oboe:trace   Turn on trace mode
   
  srv_id=_ri:oboe:strict   Be strict in parsing
  srv_id=_ri:oboe:no_strict   Don't be strict in parsing, eg supply admin version and namespace IDs if omitted
  srv_id=_ri:oboe:validate   Validate
  srv_id=_ri:oboe:no_validate   Don't validate
  srv_id=_ri:oboe:parse_uris   Parse all URI strings to check for well-formedness
  srv_id=_ri:oboe:no_parse_uris   Do not parse URI strings
  srv_id=_ri:oboe:refs   Resolve references to external resources
  srv_id=_ri:oboe:no_refs   Don't resolve references to external resources
   
  srv_id=_ri:oboe:forward   Forward OpenURL to another service identified with a resolver identifier
  srv_id=_ri:oboe:keep_context   Keep current context when forwarding to another service, but allow additions
  srv_id=_ri:oboe:freeze_context   Preserve current context when forwarding to another service, ie no additions/deletions
   
  srv_id=_ri:oboe:to_s   Render OpenURL as a property list
  srv_id=_ri:oboe:to_excel   Render OpenURL as an Excel spreadsheet
  srv_id=_ri:oboe:to_query   Render OpenURL as a URL querystring
  srv_id=_ri:oboe:to_rdf   Render OpenURL as an RDF description
  srv_id=_ri:oboe:to_xml   Render OpenURL as an XML document
